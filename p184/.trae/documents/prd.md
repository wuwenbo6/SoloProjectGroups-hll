## 1. 产品概述

gRPC 动态调用平台 —— 一个通过 gRPC Server Reflection 自动发现服务方法签名、动态构造请求并调用任意 gRPC 服务的 Web 工具。面向开发者与运维人员，解决手动编写 gRPC 客户端代码的低效问题，实现零代码即可调试和调用 gRPC 服务。

- 目标用户：后端开发者、SRE/运维工程师、测试工程师
- 核心价值：无需编写客户端代码即可即时调用任意 gRPC 服务，大幅提升调试效率

## 2. 核心功能

### 2.1 功能模块

1. **服务发现页**：输入 gRPC 服务地址，通过 Server Reflection 自动列出所有服务及其方法
2. **方法调用页**：选择具体方法，查看请求/响应签名，输入 JSON 参数，发起调用并查看结果

### 2.2 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|---------|---------|---------|
| 服务发现页 | 连接配置 | 输入 gRPC 服务地址（host:port），支持明文/TLS 切换，点击连接 |
| 服务发现页 | 服务列表 | 展示反射获取的所有服务名，点击展开显示方法列表 |
| 服务发现页 | 方法签名 | 选中方法后展示完整请求/响应 protobuf 类型签名 |
| 方法调用页 | JSON 编辑器 | 可编辑的 JSON 输入区域，支持语法高亮与校验 |
| 方法调用页 | 调用控制 | 发起调用、取消调用按钮，超时时间设置 |
| 方法调用页 | 响应展示 | 展示 JSON 格式的响应数据、错误信息、调用耗时 |

## 3. 核心流程

用户输入 gRPC 服务地址 → 后端通过 Server Reflection 获取服务列表 → 用户选择服务和方法 → 后端解析方法签名（请求/响应类型）→ 前端展示 JSON 模板 → 用户编辑 JSON 参数 → 发起调用 → 后端动态构造 gRPC 请求并转发 → 返回 JSON 响应

```mermaid
flowchart TD
    "A[用户输入 gRPC 地址]" --> "B[后端发起 Server Reflection]"
    "B" --> "C[返回服务列表及方法签名]"
    "C" --> "D[用户选择方法]"
    "D" --> "E[后端解析请求类型生成 JSON 模板]"
    "E" --> "F[用户编辑 JSON 参数]"
    "F" --> "G[提交调用请求]"
    "G" --> "H[后端动态构造 gRPC 请求]"
    "H" --> "I[返回 JSON 响应/错误]"
```

## 4. 用户界面设计

### 4.1 设计风格

- 主色调：深色科技风，暗色背景 + 青绿色（teal/cyan）强调色
- 辅助色：灰色系用于面板与卡片背景，红色用于错误状态
- 按钮风格：圆角、微渐变、hover 发光效果
- 字体：代码区域使用 JetBrains Mono，UI 区域使用 Noto Sans SC
- 布局风格：左侧导航 + 主内容区，卡片式面板

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI 元素 |
|---------|---------|---------|
| 服务发现页 | 连接配置 | 输入框、TLS 开关、连接按钮 |
| 服务发现页 | 服务列表 | 可折叠树形结构，服务名+方法名 |
| 服务发现页 | 方法签名 | 代码风格展示区域，类型高亮 |
| 方法调用页 | JSON 编辑器 | 等宽字体文本编辑区，行号显示 |
| 方法调用页 | 调用控制 | 超时输入、调用/取消按钮、状态指示 |
| 方法调用页 | 响应展示 | JSON 格式化展示，状态码+耗时标签 |

### 4.3 响应式

- 桌面优先设计，左右分栏布局
- 平板适配：服务列表折叠为侧边抽屉
- 移动端：单列布局，切换面板展示
