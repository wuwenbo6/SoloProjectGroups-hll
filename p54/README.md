# 眼动追踪实验平台

基于 webgazer.js 的眼动追踪问卷实验平台，支持实时眼动数据记录、热点图生成、AOI兴趣区分析和多被试对比。

## 功能特性

### 1. 眼动数据采集
- 集成 webgazer.js 进行实时眼动追踪
- 9点校准确保数据准确性
- 实时记录注视点坐标和时间戳

### 2. 问卷实验系统
- 4道标准化问卷题目
- 支持被试信息录入（编号、年龄、性别）
- 实时追踪状态显示

### 3. 数据可视化
- **热点图**：展示注视点分布密度
- **时序注视序列**：展示Y坐标随时间变化
- **AOI分析图表**：各兴趣区停留时间柱状图

### 4. AOI兴趣区分析
- 4个预设兴趣区（对应4道题目）
- 计算指标：
  - 总停留时间
  - 注视次数
  - 进入/退出次数
  - 首次进入时间
  - 平均停留时间
  - 时间占比

### 5. 多被试对比
- 支持选择多个被试进行对比
- AOI停留时间对比图表
- 扫描路径对比可视化
- 统计数据汇总表格

### 6. 数据存储
- SQLite 数据库存储
- 被试信息表
- 实验数据表（含JSON格式的眼动数据和分析结果）

## 安装与运行

### 1. 安装Python依赖

```bash
pip install -r requirements.txt
```

### 2. 启动后端服务器

```bash
cd backend
python app.py
```

服务器将在 `http://localhost:5000` 启动。

### 3. 访问应用

在浏览器中打开 `http://localhost:5000`

## 使用流程

### 实验阶段

1. **填写被试信息**：输入被试编号、年龄、性别
2. **眼动校准**：点击屏幕上9个校准点完成校准
3. **填写问卷**：阅读并回答4道问题（眼动数据同步记录）
4. **提交数据**：点击提交保存实验数据

### 结果分析阶段

1. 点击"结果分析"标签
2. 选择被试ID
3. 点击"加载结果"查看：
   - 注视热点图
   - 时序注视序列图
   - AOI停留时间分析
   - 详细统计表格

### 多被试对比阶段

1. 点击"被试对比"标签
2. 按住Ctrl键选择多个被试
3. 点击"生成对比报告"查看：
   - AOI停留时间对比图
   - 统计数据对比表
   - 扫描路径对比

## API接口

### 获取被试列表
```
GET /api/subjects
```

### 保存实验数据
```
POST /api/experiment
Content-Type: application/json

{
  "subject_id": "S001",
  "age": 25,
  "gender": "male",
  "answers": {"q1": "5", "q2": "4", ...},
  "gaze_data": [{"timestamp": 0, "x": 500, "y": 300}, ...],
  "total_time": 30000
}
```

### 获取单个被试实验数据
```
GET /api/experiment/<subject_id>
```

### 多被试对比
```
POST /api/comparison
Content-Type: application/json

{
  "subject_ids": ["S001", "S002", "S003"]
}
```

### 获取统计信息
```
GET /api/stats
```

## AOI定义

| AOI ID | 名称 | 位置（%） | 尺寸（%） |
|--------|------|-----------|-----------|
| q1 | 问题1 - 满意度 | (0, 0) | 100x20 |
| q2 | 问题2 - 易用性 | (0, 20) | 100x20 |
| q3 | 问题3 - 推荐意愿 | (0, 40) | 100x20 |
| q4 | 问题4 - 理解程度 | (0, 60) | 100x40 |

## 技术栈

- **前端**：HTML5, CSS3, Vanilla JavaScript
- **眼动追踪**：webgazer.js
- **后端**：Python Flask
- **数据库**：SQLite
- **可视化**：Canvas API

## 注意事项

1. 使用Chrome或Firefox浏览器获得最佳效果
2. 确保摄像头权限已开启
3. 校准时光线充足效果更好
4. 保持头部相对稳定以获得更准确的追踪数据
