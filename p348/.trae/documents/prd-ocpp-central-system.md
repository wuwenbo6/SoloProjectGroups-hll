## 1. 产品概述

OCPP（开放充电点协议）中央系统，用于管理和监控电动汽车充电桩，处理充电桩注册、充电会话生命周期管理及费用计算，为运营管理者提供可视化的数据展示平台。

- **主要用途**：作为充电桩网络的核心控制中心，实现与充电桩的双向通信，管理充电会话，计算充电费用
- **解决问题**：标准化充电桩与中央系统的通信，实现充电过程的自动化管理和费用结算
- **目标用户**：充电桩运营商、充电设施管理员、财务人员
- **产品价值**：提高充电桩网络运营效率，降低人工管理成本，提供准确的费用结算和数据分析

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 系统管理员 | 系统预置账号 | 查看所有充电会话、费用明细，管理充电桩配置，查看统计报表 |
| 运营人员 | 管理员创建 | 查看充电会话状态，处理异常订单 |

### 2.2 功能模块

1. **OCPP通信服务**：WebSocket服务、SOAP服务，支持OCPP 1.6协议
2. **充电会话管理**：BootNotification注册、StartTransaction开始充电、StopTransaction结束充电
3. **费用计算引擎**：按充电时长/电量计算费用，支持阶梯电价
4. **数据展示前端**：充电会话列表、费用明细展示、实时状态监控

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|---------|----------|----------|
| 仪表盘 | 概览卡片 | 展示在线充电桩数量、当前充电会话数、今日充电量、今日收入 |
| 仪表盘 | 实时状态 | 充电桩在线/离线状态、充电中/空闲状态 |
| 充电会话 | 会话列表 | 展示所有充电会话，支持按时间、充电桩、状态筛选 |
| 充电会话 | 会话详情 | 展示单条会话的完整信息，包括开始/结束时间、充电量、费用明细 |
| 费用明细 | 费用列表 | 按会话展示费用构成，包括电费、服务费、总金额 |
| 充电桩管理 | 充电桩列表 | 展示已注册的充电桩信息，包括型号、固件版本、最后心跳时间 |

## 3. 核心流程

### 3.1 充电桩注册流程
充电桩启动后向中央系统发送BootNotification请求，系统验证充电桩身份后返回注册结果和心跳间隔。

```mermaid
flowchart LR
    A["充电桩启动"] --> B["发送BootNotification"]
    B --> C["中央系统验证"]
    C --> D["返回Accepted/Rejected"]
    D --> E["设置心跳间隔"]
    E --> F["充电桩开始心跳"]
```

### 3.2 充电会话流程
用户刷卡后，充电桩发送StartTransaction开始充电，充电结束后发送StopTransaction，系统计算费用。

```mermaid
flowchart LR
    A["用户刷卡"] --> B["发送StartTransaction"]
    B --> C["系统创建会话记录"]
    C --> D["返回transactionId"]
    D --> E["开始充电"]
    E --> F["用户结束充电"]
    F --> G["发送StopTransaction"]
    G --> H["系统计算费用"]
    H --> I["更新会话状态"]
    I --> J["返回费用确认"]
```

### 3.3 费用计算流程
根据充电时长、充电电量、电价规则计算总费用。

```mermaid
flowchart LR
    A["获取StopTransaction数据"] --> B["计算充电时长"]
    B --> C["计算充电电量"]
    C --> D["匹配电价规则"]
    D --> E["计算电费 = 电量 × 电价"]
    E --> F["计算服务费"]
    F --> G["总费用 = 电费 + 服务费"]
    G --> H["保存费用明细"]
```

## 4. 用户界面设计

### 4.1 设计风格
- **主色调**：深蓝色（#1E3A8A）代表科技感和可靠性，搭配青绿色（#0D9488）作为成功/在线状态色
- **辅助色**：橙色（#EA580C）用于充电中状态，灰色（#6B7280）用于离线/空闲状态
- **按钮风格**：圆角矩形，带微妙阴影，hover时有轻微上浮效果
- **字体**：标题使用"Space Grotesk"无衬线字体，正文使用"Inter"字体
- **布局风格**：卡片式布局，清晰的信息层级，左侧导航栏 + 右侧内容区
- **图标风格**：使用Lucide图标库，线性风格，统一图标大小

### 4.2 页面设计概述

| 页面名称 | 模块名称 | UI元素 |
|---------|----------|--------|
| 仪表盘 | 概览卡片 | 四个统计卡片横向排列，图标+数值+同比变化，渐变背景 |
| 仪表盘 | 实时状态 | 充电桩状态网格，每个卡片显示充电桩ID、状态指示灯、当前功率 |
| 充电会话 | 会话列表 | 数据表格，支持分页和筛选，每行显示会话摘要信息 |
| 充电会话 | 会话详情 | 抽屉式详情面板，展示完整信息和时间线 |
| 费用明细 | 费用列表 | 卡片式列表，每张卡片显示费用构成条形图 |
| 充电桩管理 | 充电桩列表 | 表格展示充电桩信息，状态标签使用不同颜色 |

### 4.3 响应式设计
- **设计优先级**：桌面端优先，适配1280px及以上屏幕
- **平板适配**：1024px时导航栏折叠为图标模式，统计卡片改为2×2布局
- **手机适配**：768px以下采用垂直流式布局，表格改为卡片式展示
- **触控优化**：移动端按钮最小高度44px，增加触摸反馈效果

### 4.4 动效设计
- 页面加载时采用渐进式入场动画，元素从下往上渐入
- 数据更新时有平滑的数值过渡动画
- 状态变化时添加闪烁提示效果
- 卡片hover时有轻微上浮和阴影加深效果
