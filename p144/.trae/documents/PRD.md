## 1. 产品概述
Web模拟器用于模拟多用户无线通信场景下的资源块(RU)分配过程，根据用户信道质量动态分配资源并计算系统吞吐量。支持公平调度与最大吞吐量两种经典算法的对比分析，为通信算法研究和教学提供可视化工具。

- 目标用户：通信工程研究者、学生、算法工程师
- 核心价值：直观展示无线资源分配过程，量化对比不同调度算法的性能差异

## 2. 核心功能

### 2.1 功能模块

1. **主模拟器页面**：参数配置面板、资源块着色图、吞吐量统计、算法对比

### 2.2 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|-----------|-------------|---------------------|
| 模拟器主页 | 参数配置面板 | 设置用户数量、总RB数、信道模型、信噪比范围、运行时隙数 |
| 模拟器主页 | 资源块着色图 | 可视化展示每个时隙各用户占用的资源块，不同用户用不同颜色区分 |
| 模拟器主页 | 吞吐量统计 | 实时显示每个用户的吞吐量、总吞吐量、公平性指数 |
| 模拟器主页 | 算法对比面板 | 并列展示公平调度和最大吞吐算法的资源分配结果和性能指标 |
| 模拟器主页 | 运行控制 | 开始/暂停/重置模拟，单步执行，速度调节 |

## 3. 核心流程

```mermaid
flowchart LR
    A["配置模拟参数"] --> B["初始化多用户信道"]
    B --> C["选择调度算法"]
    C --> D["计算信道质量(SNR)"]
    D --> E["分配资源块(RU)"]
    E --> F["计算时隙吞吐量"]
    F --> G["更新统计数据"]
    G --> H["渲染资源着色图"]
    H --> I["是否继续?"]
    I -->|是| D
    I -->|否| J["输出最终对比报告"]
```

**流程说明**：
1. 用户配置模拟参数（用户数、RB数、时隙数等）
2. 系统初始化每个用户的信道模型
3. 按选择的算法（公平/最大吞吐）进行调度
4. 每个时隙：计算SNR → 分配RB → 计算吞吐量 → 更新统计 → 渲染可视化
5. 模拟结束后输出两种算法的性能对比报告

## 4. 用户界面设计

### 4.1 设计风格
- **主色调**：深蓝 (#0ea5e9) 作为科技感主色，配合深灰 (#0f172a) 背景
- **辅助色**：翠绿 (#10b981) 代表公平算法，橙红 (#f97316) 代表最大吞吐算法
- **数据可视化**：10种高区分度的用户配色方案
- **字体**：Monospace字体用于数据展示，确保数字对齐
- **布局**：三栏式布局（左：参数配置 | 中：资源着色图 | 右：统计对比）
- **动效**：时隙切换时的平滑过渡动画，资源分配时的渐入效果

### 4.2 页面设计概述

| 页面名称 | 模块名称 | UI元素 |
|-----------|-------------|-------------|
| 模拟器主页 | 参数配置面板 | 滑块、数字输入框、下拉选择器、标签式分组 |
| 模拟器主页 | 资源块着色图 | SVG网格热力图、悬停提示信息、时隙滚动条 |
| 模拟器主页 | 吞吐量统计 | 实时折线图、数据卡片、进度条 |
| 模拟器主页 | 算法对比面板 | 并列双栏布局、性能雷达图、指标对比表格 |
| 模拟器主页 | 运行控制 | 播放/暂停/重置按钮、速度调节滑块 |

### 4.3 响应式
- 桌面端（默认）：三栏并列完整布局
- 平板端：参数配置折叠为侧边抽屉，统计区置于底部
- 移动端：单栏垂直滚动布局，触控友好的滑块和按钮

### 4.4 视觉细节
- 资源块采用圆角矩形，相邻RB间有细微间隙
- 用户颜色具有良好的可区分性和色盲友好性
- 信道质量通过颜色饱和度/亮度动态体现
- 悬停在RB上显示详细信息：用户ID、SNR、MCS、吞吐量
