## 1. 产品概述

SNMP Trap 监控面板是一个实时 SNMP Trap 接收与可视化系统，支持 SNMP v2c 和 v3 协议，为网络管理员提供直观的 Trap 事件监控与原始报文分析能力。

- 解决网络运维中 Trap 事件难以实时追踪和快速定位的问题
- 面向网络工程师和运维人员，提供轻量级、开箱即用的 Trap 监控工具

## 2. 核心功能

### 2.1 功能模块

1. **Trap 监控页面**：实时 Trap 列表、变量绑定解析、原始报文查看、协议版本筛选
2. **配置管理页面**：SNMP 监听配置（端口、v2c 社区字符串、v3 凭证）、连接状态监控

### 2.2 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| Trap 监控页面 | 实时 Trap 列表 | 展示收到的所有 Trap，按时间倒序排列，显示时间戳、源IP、Trap OID、协议版本 |
| Trap 监控页面 | 协议版本筛选 | 支持按 v2c / v3 筛选 Trap 记录 |
| Trap 监控页面 | 变量绑定详情 | 展开单条 Trap，显示所有 Variable Bindings（OID + 值） |
| Trap 监控页面 | 原始报文查看 | 以十六进制/结构化格式展示 Trap 原始 PDU 报文 |
| Trap 监控页面 | 清空与导出 | 支持清空当前 Trap 列表、导出为 JSON |
| 配置管理页面 | 监听配置 | 配置监听端口、v2c 社区字符串、v3 用户名/认证/加密参数 |
| 配置管理页面 | 服务状态 | 显示监听服务运行状态、已接收 Trap 计数 |

## 3. 核心流程

用户启动后端 SNMP Trap 监听服务 → 网络设备发送 Trap → 后端 pysnmp 接收并解析 → 通过 WebSocket 实时推送到前端 → 前端更新 Trap 列表 → 用户点击查看变量绑定和原始报文

```mermaid
flowchart TD
    "A[网络设备发送 SNMP Trap]" --> "B[后端 pysnmp 接收 Trap]"
    "B" --> "C{协议版本?}"
    "C" --> "|v2c| D[使用社区字符串验证解析]"
    "C" --> "|v3| E[使用 USM 认证/加密解析]"
    "D" --> "F[提取变量绑定 OID+Value]"
    "E" --> "F"
    "F" --> "G[保存原始报文与解析数据]"
    "G" --> "H[通过 WebSocket 推送前端]"
    "H" --> "I[前端实时更新 Trap 列表]"
    "I" --> "J[用户点击查看详情]"
    "J" --> "K[展示变量绑定和原始报文]"
```

## 4. 用户界面设计

### 4.1 设计风格

- 主色调：深色科技风（#0a0e17 深蓝黑底色 + #00e5a0 青绿色强调色）
- 按钮：圆角微发光按钮，hover 时发光加强
- 字体：JetBrains Mono（数据展示）+ Noto Sans SC（中文UI）
- 布局：左侧导航 + 右侧内容区，顶部状态栏
- 图标：线性图标风格，匹配科技监控主题

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| Trap 监控页面 | 顶部状态栏 | 在线状态指示灯、Trap 计数器、监听端口显示 |
| Trap 监控页面 | 筛选工具栏 | v2c/v3 切换标签、搜索框、清空/导出按钮 |
| Trap 监控页面 | Trap 列表 | 时间戳、源IP、Trap OID、版本标签，hover 高亮行 |
| Trap 监控页面 | 变量绑定面板 | 展开式详情，OID 与值分列显示，语法高亮 |
| Trap 监控页面 | 原始报文面板 | 十六进制格式化显示，结构化字段标注 |
| 配置管理页面 | 配置表单 | 端口输入、v2c 社区字符串、v3 凭证分组表单 |
| 配置管理页面 | 状态面板 | 监听状态指示、统计信息卡片 |

### 4.3 响应式设计

- 桌面优先设计，最小宽度 1024px
- Trap 列表在小屏下隐藏部分列，保留关键信息
- 变量绑定和原始报文在移动端以全屏抽屉方式展示
