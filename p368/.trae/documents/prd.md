## 1. 产品概述

NVMe/TCP双路径目标器模拟器——一个可视化展示NVMe/TCP存储网络多路径故障切换的交互式仪表盘。模拟两条独立网络路径（PathA/PathB），客户端同时连接双路径，当活动路径断开时自动无缝切换到备用路径，确保IO不中断，并实时呈现路径状态、切换事件和IO流量。

- 目标用户：存储网络工程师、系统管理员、NVMe-oF技术学习者
- 核心价值：直观理解NVMe多路径故障切换机制，无需真实硬件即可体验路径冗余效果

## 2. 核心功能

### 2.1 功能模块

1. **仪表盘页面**：路径状态总览、实时IO流量、切换事件日志、控制面板

### 2.2 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 仪表盘 | 路径状态卡片 | 展示PathA/PathB连接状态（在线/离线）、当前活动路径高亮标识、路径延迟模拟值 |
| 仪表盘 | 切换计数器 | 显示累计路径切换次数、最近一次切换时间、切换方向（A→B 或 B→A） |
| 仪表盘 | IO流量图 | 实时折线图展示双路径IO吞吐量（读写IOPS/带宽），切换时IO不中断的可视化 |
| 仪表盘 | 事件日志 | 时间线展示路径切换、断连、恢复等事件，含时间戳和事件类型标签 |
| 仪表盘 | 控制面板 | 手动断开/恢复指定路径按钮、自动故障注入开关、IO负载调节滑块 |

## 3. 核心流程

用户通过控制面板模拟路径故障，系统自动执行故障检测和路径切换，前端实时展示整个过程。


## 4. 用户界面设计

### 4.1 设计风格

- 主色调：深色赛博工业风——深蓝灰底色（#0a0e1a），霓虹青色（#00f0ff）作为主强调色，警戒橙（#ff6b35）作为故障/切换色
- 按钮：圆角胶囊型按钮，在线状态为霓虹青发光效果，离线状态为暗灰+橙红描边
- 字体：显示字体使用 JetBrains Mono（技术/终端风格），UI字体使用 Source Sans 3
- 布局：左侧路径状态面板 + 中间IO流量图 + 右侧事件日志，底部控制面板
- 图标/装饰：数据流动画粒子、脉冲环表示活动路径、网格线背景

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 仪表盘 | 路径状态卡片 | 双卡片布局，活动路径有脉冲光晕动画，状态指示灯（绿/红），延迟数值 |
| 仪表盘 | 切换计数器 | 大号数字显示切换次数，箭头动画指示切换方向，最近切换时间 |
| 仪表盘 | IO流量图 | Canvas折线图，双线叠加（PathA青色/PathB蓝色），切换点标记竖线 |
| 仪表盘 | 事件日志 | 垂直时间线，彩色标签（连接/断开/切换/恢复），自动滚动最新 |
| 仪表盘 | 控制面板 | PathA/PathB断开/恢复按钮组，自动故障注入开关，IO强度滑块 |

### 4.3 响应式

桌面优先设计，最小支持1280px宽度。在窄屏下，三栏布局切换为单栏纵向堆叠。
