## 1. 产品概述

Swift 对象存储清理助手 —— 基于 Python swiftclient 的 OpenStack Swift 存储清理工具，自动遍历容器与对象，通过 X-Timestamp 识别超过 90 天未访问的对象，前端提供可视化清理列表与一键删除功能。

- 面向运维人员与存储管理员，解决对象存储空间浪费问题
- 自动化识别冷数据，降低存储成本，提升管理效率

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 存储管理员 | 配置 Swift 连接信息 | 遍历容器/对象、查看冷数据、执行删除操作 |

### 2.2 功能模块

1. **概览页**：Swift 连接配置、存储统计概览、扫描触发
2. **清理列表页**：超过 90 天未访问的对象列表、筛选排序、一键删除

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 概览页 | 连接配置 | 输入 Swift Auth URL、用户名、密码、项目等信息，测试连接 |
| 概览页 | 存储统计 | 显示总容器数、总对象数、冷数据数量、冷数据占比 |
| 概览页 | 扫描控制 | 触发全量扫描、显示扫描进度、上次扫描时间 |
| 清理列表页 | 对象列表 | 按容器分组展示冷数据对象，显示名称、大小、最后访问时间、容器归属 |
| 清理列表页 | 筛选排序 | 按容器筛选、按大小/时间排序、搜索对象名 |
| 清理列表页 | 删除操作 | 单个删除、批量选择删除、一键全部清理，操作前二次确认 |

## 3. 核心流程

用户配置 Swift 连接信息 → 触发扫描 → 后端遍历所有容器和对象 → 根据 X-Timestamp 计算未访问天数 → 前端展示超过 90 天的对象列表 → 用户选择对象执行删除 → 后端调用 swiftclient 删除对象 → 刷新列表

```mermaid
flowchart TD
    "A[配置 Swift 连接]" --> "B[触发扫描]"
    "B" --> "C[后端遍历容器和对象]"
    "C" --> "D[读取 X-Timestamp 计算未访问天数]"
    "D" --> "E{超过 90 天?}"
    "E" --> "是[标记为冷数据]"
    "E" --> "否[跳过]"
    "标记为冷数据" --> "F[前端展示清理列表]"
    "F" --> "G[用户选择删除]"
    "G" --> "H[二次确认]"
    "H" --> "I[后端调用 swiftclient 删除]"
    "I" --> "J[刷新列表]"
```

## 4. 用户界面设计

### 4.1 设计风格

- 主色调：深青色 (#0F766E) 搭配暖橙色 (#F97316) 作为警告/删除强调色
- 按钮风格：圆角矩形，删除按钮使用红色危险样式
- 字体：DM Sans 作为主字体，JetBrains Mono 用于数据展示
- 布局风格：左侧导航栏 + 右侧内容区，卡片式数据展示
- 图标风格：Lucide 线性图标

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI 元素 |
|----------|----------|---------|
| 概览页 | 连接配置 | 表单卡片，输入框 + 测试连接按钮，深色卡片背景 |
| 概览页 | 存储统计 | 四宫格统计卡片，图标 + 数字 + 标签，动画数字递增效果 |
| 概览页 | 扫描控制 | 大按钮 + 进度条 + 状态文字，脉冲动画 |
| 清理列表页 | 对象列表 | 表格组件，行悬停高亮，左侧复选框，容器分组折叠 |
| 清理列表页 | 筛选排序 | 顶部工具栏，下拉筛选 + 搜索框 + 排序按钮 |
| 清理列表页 | 删除操作 | 底部浮动操作栏，批量删除按钮 + 选中计数，确认弹窗 |

### 4.3 响应式

桌面优先设计，最小支持 1024px 宽度，内容区自适应

### 4.4 3D 场景

不适用
