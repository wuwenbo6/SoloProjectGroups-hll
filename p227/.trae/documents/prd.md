## 1. 产品概述
HL7检验结果管理平台，用于接收、解析、存储和展示HL7 v2.x格式的医学检验结果消息。通过TCP或文件方式接收HL7消息，自动解析关键段（MSH/PID/OBR/OBX），存入数据库，并提供直观的Web界面供医护人员查询和浏览检验结果。
- 解决医疗机构中HL7消息的接收与解析问题，将非结构化的HL7消息转化为可查询、可浏览的结构化数据
- 目标用户为医院检验科、信息科技术人员及临床医生

## 2. 核心功能

### 2.1 用户角色
| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 操作员 | 默认登录 | 查看检验结果、搜索患者、查看消息日志 |
| 管理员 | 默认登录 | 操作员权限 + 查看系统状态、管理HL7连接 |

### 2.2 功能模块
1. **检验结果总览页**: 患者列表、最近检验结果概览、搜索功能
2. **患者检验详情页**: 患者基本信息、检验订单列表、检验结果明细（含参考范围标注）
3. **消息监控页**: HL7消息接收日志、解析状态、原始消息查看
4. **系统状态页**: TCP服务器状态、数据库连接状态、消息统计

### 2.3 页面详情
| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 检验结果总览页 | 搜索栏 | 按患者姓名、ID搜索 |
| 检验结果总览页 | 患者列表 | 展示最近接收的患者及其最新检验结果摘要 |
| 检验结果总览页 | 统计卡片 | 今日消息数、患者数、异常结果数、待审核数 |
| 患者检验详情页 | 患者信息卡 | PID段解析的患者基本信息（姓名、ID、性别、出生日期） |
| 患者检验详情页 | 检验订单列表 | OBR段解析的检验订单（订单号、检验项目、申请时间） |
| 患者检验详情页 | 检验结果表 | OBX段解析的检验结果（项目名、值、单位、参考范围、异常标记） |
| 患者检验详情页 | 原始消息查看 | 查看原始HL7消息文本 |
| 消息监控页 | 消息列表 | 按时间倒序展示接收的HL7消息，含解析状态标记 |
| 消息监控页 | 消息详情 | 点击查看单条消息的解析结果与原始内容 |
| 系统状态页 | 服务状态 | TCP服务器运行状态、端口、连接数 |
| 系统状态页 | 数据库状态 | 数据库连接状态、表记录数 |

## 3. 核心流程

用户打开系统 → 查看检验结果总览 → 搜索或选择患者 → 查看患者检验详情 → 查看具体检验结果（含异常标记）→ 可选查看原始HL7消息。

HL7消息流程：外部系统发送HL7消息 → TCP服务器接收（或文件导入）→ 解析MSH/PID/OBR/OBX段 → 存入数据库 → 前端实时刷新展示。

## 4. 用户界面设计

### 4.1 设计风格
- 主色调：深蓝 (#1e3a5f) + 青色 (#0ea5e9) 渐变，象征医疗与科技
- 辅助色：白色/浅灰背景，异常结果用红色 (#ef4444) 高亮，正常用绿色 (#22c55e)
- 按钮风格：圆角 (rounded-lg)，轻微阴影，hover时加深
- 字体：标题用 DM Sans，正文用系统字体栈
- 布局风格：左侧导航栏 + 右侧内容区，卡片式内容展示
- 图标：Lucide图标库

### 4.2 页面设计概览
| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 检验结果总览页 | 统计卡片 | 四张卡片横排，深色背景白色文字，数字加粗大号 |
| 检验结果总览页 | 搜索栏 | 圆角输入框 + 搜索图标，占满宽度 |
| 检验结果总览页 | 患者列表 | 表格布局，行hover高亮，异常值红色标签 |
| 患者检验详情页 | 患者信息卡 | 左侧头像占位 + 右侧信息网格 |
| 患者检验详情页 | 检验结果表 | 表格，异常值行背景淡红，参考范围列灰色小字 |
| 患者检验详情页 | 原始消息 | 可展开的代码块，等宽字体，深色背景 |
| 消息监控页 | 消息列表 | 表格 + 状态标签（成功/失败/解析中） |
| 系统状态页 | 状态面板 | 圆形指示灯 + 状态文字 + 详情卡片 |

### 4.3 响应式
- 桌面优先设计，导航栏在移动端折叠为汉堡菜单
- 表格在小屏幕上转为卡片布局
- 统计卡片在移动端两列排列
