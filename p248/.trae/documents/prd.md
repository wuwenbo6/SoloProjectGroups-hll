## 1. 产品概述

DASH-IF IOP MPD 验证工具是一款面向流媒体工程师和开发者的在线检测平台，用于上传 MPEG-DASH 的 MPD（Media Presentation Description）文件，自动检查其是否符合 DASH-IF IOP（Interoperability Points）规范，包括 SegmentTemplate 模板填充、timescale 时间尺度、Representation 属性完整性等关键规则，并以清晰的报告形式输出合规结果与警告信息。

- 解决问题：手动审查 MPD 文件繁琐且易遗漏，自动化验证可大幅提升效率和准确性
- 目标用户：流媒体平台工程师、视频编码与封装开发者、DASH 标准测试人员

## 2. 核心功能

### 2.1 用户角色
| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 访客用户 | 无需注册 | 上传 MPD 文件、查看验证报告 |

### 2.2 功能模块
1. **验证页面**：MPD 文件上传、实时验证进度、验证报告展示、警告与错误详情

### 2.3 页面详情
| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 验证页面 | 文件上传区 | 拖拽或点击上传 MPD 文件，支持 .mpd / .xml 格式，文件大小上限 10MB |
| 验证页面 | 验证进度 | 上传后显示验证进度状态（解析中→验证中→完成） |
| 验证页面 | 报告概览 | 展示验证总分、通过/警告/错误数量统计 |
| 验证页面 | 规则检查列表 | 逐条展示 DASH-IF IOP 规则检查结果，支持按严重级别筛选（ERROR / WARNING / INFO） |
| 验证页面 | 详情面板 | 点击某条规则可展开查看具体问题描述、涉及的 XML 元素路径及修复建议 |
| 验证页面 | MPD 源码预览 | 侧栏展示原始 MPD XML 内容，高亮标注问题行 |

## 3. 核心流程

用户打开验证页面 → 上传 MPD 文件 → 后端接收文件并解析 XML → 执行 DASH-IF IOP 规则检查 → 生成验证报告 → 前端实时展示结果

```mermaid
    "A[用户上传MPD文件]" --> "B[后端接收并解析XML]"

    "C" --> "|是| E[执行DASH-IF IOP规则检查]"

    "E" --> "I[Period连续性验证]"
    "E" --> "J[其他IOP规则验证]"
    "F" --> "K[汇总验证结果]"
    "G" --> "K"
    "H" --> "K"
    "I" --> "K"
    "J" --> "K"
    "K" --> "L[生成验证报告]"
    "L" --> "M[前端展示报告与警告]"
```

## 4. 用户界面设计

### 4.1 设计风格
- 主色调：深蓝灰色（#1a1d23）背景 + 青绿色（#00e5a0）强调色
- 辅助色：琥珀色（#f59e0b）用于 WARNING、红色（#ef4444）用于 ERROR
- 按钮风格：圆角（8px）、微透明背景、hover 发光效果
- 字体：JetBrains Mono（代码/数据展示）+ Outfit（界面文字）
- 布局风格：居中单栏布局，报告区域卡片式排列
- 图标风格：Lucide 线性图标

### 4.2 页面设计概览
| 页面名称 | 模块名称 | UI 元素 |
|----------|----------|---------|
| 验证页面 | 文件上传区 | 虚线边框拖拽区域、上传图标、文件名显示、上传按钮；深色背景、青绿边框高亮 |
| 验证页面 | 验证进度 | 步骤指示器（解析→验证→完成），动画过渡 |
| 验证页面 | 报告概览 | 三个统计卡片（通过/警告/错误），数字动画递增效果，环形进度图 |
| 验证页面 | 规则检查列表 | 可折叠列表项，左侧色条标识严重级别，右侧状态标签 |
| 验证页面 | 详情面板 | 滑出式面板，显示问题描述、XML 路径、修复建议 |
| 验证页面 | MPD 源码预览 | 代码编辑器风格，语法高亮，问题行红框标注 |

### 4.3 响应式设计
- 桌面优先设计，最小宽度 1024px
- 平板适配：报告区域单列排列
- 移动端适配：上传区全宽、统计卡片横排缩小
