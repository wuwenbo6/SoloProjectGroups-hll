## 1. 产品概述

CoAP文件分块上传系统 —— 基于CoAP协议（RFC 7252）和Block1传输选项（RFC 7959）实现的文件分块上传客户端/服务器，配合Web前端实现文件选择与实时上传进度展示。

- 解决物联网/受限网络环境下大文件可靠传输的问题，通过Block1选项将文件分为小块逐块确认上传
- 面向开发者与IoT场景，演示CoAP分块传输核心机制

## 2. 核心功能

### 2.1 功能模块

1. **文件上传页面**：文件选择、上传控制、实时进度展示、传输日志
2. **传输记录页面**：历史传输记录列表、状态查看

### 2.2 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|---------|---------|---------|
| 文件上传页 | 文件选择器 | 拖拽或点击选择文件，显示文件名、大小、类型 |
| 文件上传页 | 上传控制 | 开始/暂停/取消上传按钮 |
| 文件上传页 | 进度展示 | 分块进度条、当前块/总块数、速度、预计剩余时间 |
| 文件上传页 | 传输日志 | 每块发送与确认的实时日志流，含Block1选项详情 |
| 传输记录页 | 记录列表 | 历史上传记录表格，含文件名、大小、状态、时间 |

## 3. 核心流程

用户在前端选择文件 → 前端通过HTTP API将文件发送至桥接服务 → 桥接服务中的CoAP客户端将文件按Block1规范分块 → 逐块发送至CoAP服务器 → 服务器逐块确认（2.31 Continue / 2.04 Changed） → 进度通过WebSocket实时推送至前端

```mermaid
flowchart TD
    "A[用户选择文件]" --> "B[前端发送文件至HTTP桥接]"
    "B" --> "C[桥接服务保存临时文件]"
    "C" --> "D[CoAP客户端读取文件并分块]"
    "D" --> "E[发送Block1请求(更多块=1)]"
    "E" --> "F{服务器确认}"
    "F" --> "|2.31 Continue| "G[发送下一块"]"
    "G" --> "E"
    "F" --> "|2.04 Changed| "H[上传完成"]"
    "H" --> "I[通知前端上传成功"]"
    "E" --> "|WebSocket推送进度| "J[前端更新进度条"]"
```

## 4. 用户界面设计

### 4.1 设计风格

- 主色调：深青色 (#0D9488) + 琥珀色强调 (#F59E0B)
- 按钮风格：圆角微3D，hover时阴影加深
- 字体：JetBrains Mono（日志区域）+ Outfit（UI文本）
- 布局风格：居中卡片式布局，深色主题
- 图标风格：线条型图标（lucide-react）

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|---------|---------|--------|
| 文件上传页 | 文件选择器 | 虚线边框拖拽区域，图标+提示文字，选中后显示文件信息卡片 |
| 文件上传页 | 上传控制 | 开始按钮(主色)、取消按钮(灰色)，上传中显示脉冲动画 |
| 文件上传页 | 进度展示 | 渐变进度条、分块网格（每块一个小方块，已确认为绿色）、统计数字 |
| 文件上传页 | 传输日志 | 等宽字体终端风格日志，不同类型消息不同颜色，自动滚动 |
| 传输记录页 | 记录列表 | 表格布局，状态标签（成功/失败/进行中），分页 |

### 4.3 响应式

- 桌面优先设计，最大宽度960px居中
- 移动端自适应：日志区域折叠、进度条简化
