## 1. 产品概述

PMIPv6 本地移动锚点（LMA）模拟器——通过可视化界面展示 PMIPv6 协议中 LMA 节点的核心行为：接收来自 MAG 的代理绑定更新（PBU），建立并维护 MN 的 IPv6 前缀与 MAG 地址的绑定缓存映射关系。

- 目标用户：网络协议研究人员、移动网络开发者、教学演示场景
- 核心价值：以交互式方式理解 PMIPv6 绑定管理机制，无需真实网络设备即可验证协议流程

## 2. 核心功能

### 2.1 功能模块

1. **绑定缓存展示页**：实时展示 LMA 中所有绑定缓存条目，支持查看、刷新、模拟 PBU 发送
2. **PBU 模拟器**：提供表单模拟 MAG 向 LMA 发送代理绑定更新，支持注册、更新、注销三种操作

### 2.2 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 绑定缓存展示页 | 绑定缓存表格 | 展示所有 BCE 条目：MN 标识、MN IPv6 前缀、MAG 地址、绑定状态、生存时间、注册时间 |
| 绑定缓存展示页 | PBU 模拟器面板 | 提供表单发送 PBU（含 MN ID、IPv6 前缀、MAG 地址、生存时间），支持注册/更新/注销操作 |
| 绑定缓存展示页 | 事件日志 | 实时显示 PBU/PBA 交互记录，含时间戳、操作类型、结果状态 |
| 绑定缓存展示页 | 网络拓扑图 | 简化展示 LMA-MAG-MN 的连接关系，随绑定变化动态更新 |

## 3. 核心流程

用户通过前端界面模拟 MAG 行为，向 LMA 发送 PBU 消息；LMA 处理 PBU 后返回 PBA 响应，同时更新绑定缓存；前端实时展示缓存变化和交互日志。

```mermaid
flowchart TD
    "用户填写 PBU 表单" --> "前端发送 POST /api/pbu"
    "前端发送 POST /api/pbu" --> "LMA 后端接收 PBU"
    "LMA 后端接收 PBU" --> "验证 PBU 参数"
    "验证 PBU 参数" --> "处理绑定更新"
    "处理绑定更新" --> "创建/更新/删除 BCE"
    "创建/更新/删除 BCE" --> "返回 PBA 响应"
    "返回 PBA 响应" --> "前端更新绑定缓存表格"
    "前端更新绑定缓存表格" --> "前端更新事件日志"
    "前端定时 GET /api/bce" --> "刷新绑定缓存展示"
```

## 4. 用户界面设计

### 4.1 设计风格

- 主色调：深色科技风（暗色背景 `#0a0f1e`，荧光青绿 `#00ffc8` 作为强调色）
- 次要色：冷灰蓝 `#1e293b`、霓虹蓝 `#3b82f6`
- 按钮风格：圆角微发光按钮，hover 时增强光晕效果
- 字体：JetBrains Mono（数据/代码区域）+ Outfit（标题/正文）
- 布局风格：单页仪表盘布局，左侧 PBU 控制面板，右侧绑定缓存表格，底部事件日志
- 图标风格：Lucide 线性图标

### 4.2 页面设计概述

| 页面名称 | 模块名称 | UI 元素 |
|----------|----------|----------|
| 绑定缓存展示页 | PBU 模拟器面板 | 深色卡片，荧光边框，输入框带发光效果，操作按钮带颜色区分（注册-青绿/注销-红色） |
| 绑定缓存展示页 | 绑定缓存表格 | 深色表格，行 hover 发光，状态标签带颜色指示（Active-绿/Expired-红/Pending-黄） |
| 绑定缓存展示页 | 事件日志 | 终端风格滚动日志，等宽字体，时间戳+类型+结果颜色编码 |
| 绑定缓存展示页 | 网络拓扑图 | SVG 简化拓扑，LMA 居中，MAG 节点环绕，连线动态虚线流动动画 |

### 4.3 响应式设计

- 桌面优先设计，1920px 为基准宽度
- 平板端（< 1024px）PBU 面板折叠为顶部抽屉
- 移动端（< 768px）单列布局，表格改为卡片列表
