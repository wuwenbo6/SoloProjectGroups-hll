## 1. 产品概述

Art-Net 灯光控制台是一款基于Web的专业DMX512灯光控制应用，通过WebSocket实现前后端实时通信，后端将通道数据打包为Art-Net协议UDP数据包发送至灯光控制器，前端提供512通道推子控制及场景管理功能。

- 目标用户：舞台灯光师、活动策划人员、剧场技术人员
- 核心价值：用浏览器替代昂贵的硬件灯光控制台，降低专业灯光控制的门槛与成本

## 2. 核心功能

### 2.1 功能模块

1. **控制台页面**：512通道推子面板、通道分组、实时值显示、Art-Net连接状态
2. **场景管理页面**：场景保存/加载/删除、场景列表、场景预览

### 2.2 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|---------|---------|---------|
| 控制台 | 通道推子面板 | 512个垂直推子，每个对应一个DMX通道（0-255），支持拖拽调节、点击输入精确值 |
| 控制台 | 通道分组导航 | 按每32通道分组，快速定位通道区域 |
| 控制台 | 连接状态指示器 | 显示WebSocket连接状态、Art-Net目标IP/端口、数据发送状态 |
| 控制台 | 主推子 | Grand Master推子，按比例缩放所有通道输出 |
| 控制台 | 黑场按钮 | 一键将所有通道归零 |
| 控制台 | 场景快捷栏 | 顶部场景快捷切换，支持快速加载已保存场景 |
| 场景管理 | 场景列表 | 显示所有已保存场景，含名称、创建时间、通道数量 |
| 场景管理 | 保存场景 | 输入场景名称，保存当前512通道快照 |
| 场景管理 | 加载场景 | 加载场景数据到当前推子并实时发送Art-Net数据 |
| 场景管理 | 删除场景 | 删除已保存的场景 |
| 场景管理 | 场景预览 | 预览场景中非零通道的值 |

## 3. 核心流程

用户打开控制台页面，通过推子调整各通道值，前端通过WebSocket实时推送通道变化至后端，后端将数据打包为Art-Net DMX数据包并通过UDP发送到配置的灯光控制器IP。用户可将当前所有通道值保存为场景，后续快速加载恢复。

```mermaid
flowchart TD
    "用户调整推子" --> "前端更新通道值"
    "前端更新通道值" --> "WebSocket发送通道数据"
    "WebSocket发送通道数据" --> "后端接收通道变化"
    "后端接收通道变化" --> "打包Art-Net DMX数据包"
    "打包Art-Net DMX数据包" --> "UDP发送到灯光控制器"
    "前端更新通道值" --> "前端实时显示推子位置和数值"
    "用户保存场景" --> "前端发送保存请求"
    "前端发送保存请求" --> "后端存储场景到JSON"
    "用户加载场景" --> "前端请求场景数据"
    "前端请求场景数据" --> "后端返回场景数据"
    "后端返回场景数据" --> "前端恢复推子位置"
    "前端恢复推子位置" --> "WebSocket发送全量通道数据"
```

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：深黑/深灰背景（#0a0a0f），琥珀色/暖黄色强调色（#f59e0b），模拟真实灯光控制台氛围
- **辅助色**：通道活跃指示绿色（#22c55e），警告红色（#ef4444），中性灰色用于边框和分割
- **推子样式**：垂直滑块，带凹槽纹理效果，推子手柄为金属质感
- **字体**：显示字体使用 JetBrains Mono（等宽，适合数值显示），UI字体使用 DM Sans
- **布局**：左侧通道推子面板（可滚动），右侧控制区（连接状态、场景快捷栏）
- **图标风格**：线性图标（lucide-react），与专业控制台UI风格统一

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|---------|---------|--------|
| 控制台 | 通道推子面板 | 深色背景网格布局，垂直推子带刻度，通道编号标签，数值显示，分组标签 |
| 控制台 | 分组导航 | 水平标签栏，点击滚动到对应通道组 |
| 控制台 | 连接状态 | 顶部状态栏，连接指示灯（绿/红），目标IP输入框 |
| 控制台 | 主推子 | 右侧大尺寸垂直推子，带百分比显示 |
| 控制台 | 黑场按钮 | 红色警示按钮，按下确认后全通道归零 |
| 控制台 | 场景快捷栏 | 顶部水平滚动的场景缩略卡片 |
| 场景管理 | 场景列表 | 卡片网格布局，每卡片含场景名、时间、通道活跃数预览 |
| 场景管理 | 操作按钮 | 保存（琥珀色主按钮）、加载、删除（红色） |

### 4.3 响应式设计

- 桌面优先设计，推子面板在大屏上显示更多通道
- 平板适配：推子尺寸适当缩小，保持可操作性
- 触摸优化：推子支持触摸拖拽，增大触摸区域

### 4.4 动效设计

- 推子滑动平滑过渡（CSS transition）
- 连接状态变化时指示灯脉冲动画
- 场景加载时推子位置动画过渡
- 黑场按钮按下时全通道推子同时归零的级联动画
