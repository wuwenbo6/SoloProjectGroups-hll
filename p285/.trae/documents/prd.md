## 1. 产品概述

HomePlug AV 帧解析器是一个专业的电力线通信（PLC）协议分析工具，用于解析 HomePlug AV 标准的数据帧，提取关键信令信息，帮助工程师和网络管理员调试和诊断电力线网络问题。

- 目标用户：电力线通信工程师、网络调试人员、IoT 设备开发者
- 核心价值：将晦涩的二进制捕获数据转化为可视化、结构化的信令信息，降低协议分析门槛

## 2. 核心功能

### 2.1 功能模块

1. **数据上传页面**：文件拖拽/选择上传区域、支持 .bin/.cap/.pcap 等二进制格式、上传进度与状态反馈
2. **解析结果展示页面**：MAC 报头解析表、SOF 帧信息、TEI 终端标识提取、SACK 信令展示、CCo 中心协调器信息

### 2.2 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 数据上传页 | 上传区域 | 支持拖拽和点击上传二进制捕获文件，显示文件名和大小 |
| 数据上传页 | 历史记录 | 展示最近上传和解析的文件列表 |
| 解析结果页 | MAC 报头面板 | 展示解析后的 MAC 帧头字段（源/目的 TEI、帧类型等） |
| 解析结果页 | SOF 帧面板 | 展示 Start-of-Frame 信号详细字段 |
| 解析结果页 | 信令面板 | 展示 SACK 选择性确认、CCo 协调器信息 |
| 解析结果页 | 原始数据面板 | 以十六进制格式展示原始帧数据，高亮已解析字段 |

## 3. 核心流程

用户在前端上传 HomePlug AV 捕获的二进制文件 → Node.js 后端接收文件并保存 → 调用 C++ 解析器可执行程序 → C++ 解析器读取文件并解析帧结构 → 输出结构化 JSON → 后端将 JSON 返回前端 → 前端渲染解析结果

```mermaid
flowchart TD
    "A[用户上传捕获文件]" --> "B[Node.js 后端接收文件]"
    "B" --> "C[保存至临时目录]"
    "C" --> "D[调用 C++ 解析器]"
    "D" --> "E[C++ 解析二进制帧]"
    "E" --> "F[输出结构化 JSON]"
    "F" --> "G[后端返回解析结果]"
    "G" --> "H[前端渲染解析面板]"
```

## 4. 用户界面设计

### 4.1 设计风格

- 主色调：深蓝黑（#0B1120）搭配电光青（#00E5CC）作为强调色，体现通信协议分析的专业感和科技感
- 次要色调：深灰蓝（#1E293B）作为卡片背景，暗橙（#FF6B35）用于警告/错误状态
- 字体：JetBrains Mono（代码/数据展示）、Noto Sans SC（界面文字）
- 布局：顶部导航 + 左侧面板 + 主内容区域，卡片式布局
- 图标：Lucide 图标库，线性风格
- 按钮风格：圆角矩形，带微妙发光效果

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI 元素 |
|----------|----------|---------|
| 数据上传页 | 上传区域 | 虚线边框拖拽区、上传图标、文件类型提示、进度条 |
| 数据上传页 | 历史记录 | 紧凑列表、文件图标、时间戳、状态标签 |
| 解析结果页 | MAC 报头面板 | 键值对表格、TEI 高亮徽章、帧类型标签 |
| 解析结果页 | SOF 帧面板 | 字段网格、位域可视化条 |
| 解析结果页 | 信令面板 | SACK 位图可视化、CCo 状态指示灯 |
| 解析结果页 | 原始数据面板 | 十六进制查看器、行号、ASCII 列、高亮覆盖层 |

### 4.3 响应式

桌面优先设计，在 1280px 以上宽度提供完整三栏布局，平板和移动端折叠为单栏堆叠布局。

### 4.4 交互设计

- 上传区域：拖拽时边框变为电光青脉动效果
- 解析进行中：卡片骨架屏加载动画
- 解析完成：面板依次淡入展示，带有 0.15s 延迟的交错动画
- TEI 标识：带发光效果的徽章样式
- SACK 位图：每个位用小方块表示，确认位亮起电光青，未确认位暗灰
- 十六进制数据：已解析字段背景高亮，悬停显示字段含义工具提示
