## 1. 产品概述

mDNS 反射器监控面板 — 一个跨子网 mDNS 服务发现与转发的可视化平台。后端使用 Go 模拟 mDNS Reflector，将不同子网的服务记录（PTR/SRV/TXT）跨网段转发；前端以直观的仪表板展示来自多个网络的服务（打印机、AirPlay、HomeKit 等），帮助网络管理员实时掌握跨子网服务状态。

- 目标用户：家庭/企业网络管理员、IoT 运维人员
- 核心价值：让跨子网的 mDNS 服务可被发现、可监控、可管理

## 2. 核心功能

### 2.1 功能模块

1. **仪表板页面**：网络拓扑可视化、子网列表、服务统计、实时服务发现流
2. **子网详情页面**：某个子网下的所有服务列表、服务详情、记录类型分布

### 2.2 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 仪表板 | 网络拓扑图 | 展示子网之间的反射关系，节点为子网，连线为反射通道 |
| 仪表板 | 子网概览卡片 | 每个子网一张卡片：子网名称、CIDR、在线服务数、最近发现时间 |
| 仪表板 | 服务统计面板 | 饼图/条形图展示各类型服务数量（打印机、AirPlay、HomeKit 等） |
| 仪表板 | 实时发现流 | WebSocket 推送的新服务发现事件实时滚动列表 |
| 仪表板 | 反射器状态 | 反射器运行状态、转发包数、活跃接口 |
| 子网详情 | 服务列表 | 该子网下所有服务的表格：名称、类型、IP、端口、TXT 记录 |
| 子网详情 | 记录详情 | 点击某服务展开：PTR/SRV/TXT 记录原始内容 |
| 子网详情 | 服务过滤 | 按类型（打印机/AirPlay/…）和状态（在线/离线）过滤 |

## 3. 核心流程

用户打开仪表板 → 看到子网拓扑与服务统计 → 点击子网卡片进入详情 → 浏览/过滤该子网服务 → 点击服务查看记录详情。同时，实时发现流持续推送新发现的服务事件。

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：深色科技风 — 深灰/近黑背景 (#0a0e17)，搭配电青色 (#00d4ff) 作为主强调色，辅以暖琥珀色 (#ff9f1c) 标注告警/离线状态
- **次色调**：各子网用不同色彩区分 — 子网A电青、子网B翠绿、子网C琥珀、子网D紫罗兰
- **按钮风格**：圆角微3D，hover时发光效果
- **字体**：显示字体 JetBrains Mono（数据/代码感），UI字体 DM Sans
- **布局**：左侧固定导航 + 主内容区网格卡片布局
- **图标**：lucide-react 线性图标

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI 元素 |
|----------|----------|---------|
| 仪表板 | 网络拓扑图 | SVG 节点图，节点带脉冲动画，连线带流动粒子效果 |
| 仪表板 | 子网概览卡片 | 玻璃拟态卡片，微光边框，左侧色条标识子网 |
| 仪表板 | 服务统计面板 | 环形图 + 数字大字，渐变填充 |
| 仪表板 | 实时发现流 | 暗色终端风格滚动列表，新条目滑入动画，带时间戳和类型标签 |
| 仪表板 | 反射器状态 | 顶部状态栏：绿色脉冲圆点 + 运行时长 + 包计数 |
| 子网详情 | 服务列表 | 深色表格，行 hover 高亮，类型标签彩色小药丸 |
| 子网详情 | 记录详情 | 可展开行，等宽字体显示原始 DNS 记录，语法高亮 |
| 子网详情 | 服务过滤 | 顶部过滤条：类型下拉 + 状态切换 + 搜索框 |

### 4.3 响应式

- 桌面优先，1920px 为主设计宽度
- 平板（768px）下卡片堆叠，拓扑图缩放
- 手机（375px）隐藏侧边栏，底部标签导航
