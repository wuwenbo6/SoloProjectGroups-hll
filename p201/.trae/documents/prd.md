## 1. 产品概述

Kconfig Web Editor 是一个基于 Web 的内核配置编辑器，模仿 Linux menuconfig 的交互体验，让用户在浏览器中解析 Kconfig 文件、浏览配置树、勾选/取消配置项，并自动处理 `depends on` 依赖关系，最终生成标准 `.config` 文件。

- 目标用户：内核开发者、嵌入式系统工程师、需要定制 Linux 内核配置的技术人员
- 核心价值：降低内核配置门槛，提供比终端 menuconfig 更直观的可视化体验，自动处理依赖避免配置冲突

## 2. 核心功能

### 2.1 用户角色

无角色区分，单用户本地使用。

### 2.2 功能模块

1. **配置主页**：上传/选择 Kconfig 文件，展示配置树，支持交互式编辑
2. **配置详情面板**：选中配置项后显示详情（帮助文本、依赖关系、选项值）

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 配置主页 | 文件上传区 | 上传 Kconfig 文件或选择示例文件，触发解析 |
| 配置主页 | 配置树面板 | 以 menuconfig 风格展示配置树，支持展开/折叠、勾选/取消，显示依赖状态 |
| 配置主页 | 详情面板 | 显示选中配置项的类型、当前值、帮助文本、依赖链 |
| 配置主页 | 工具栏 | 搜索配置项、全部展开/折叠、导出 .config、重置配置 |
| 配置主页 | 搜索栏 | 按关键字搜索配置项，高亮匹配结果 |

## 3. 核心流程

用户上传 Kconfig 文件 → 后端解析生成配置树 → 前端展示 menuconfig 风格界面 → 用户交互式勾选/取消配置项 → 前端自动检查依赖关系（depends on） → 不满足依赖时自动灰显或取消 → 用户点击"导出" → 后端生成 .config 文件 → 下载

```mermaid
    "D" --> "I[用户点击导出]"
    "I" --> "J[后端生成 .config]"

```

## 4. 用户界面设计

### 4.1 设计风格

- **风格方向**：终端/复古工业风 —— 呼应 Linux 内核开发的极客气质
- **主色**：深色背景 `#0d1117`（类 GitHub Dark），搭配荧光绿 `#00ff41` 作为强调色（致敬经典终端）
- **次色**：琥珀黄 `#ffb000` 用于警告/依赖冲突，冷灰 `#8b949e` 用于禁用状态
- **按钮风格**：圆角 4px，紧凑型，扁平化设计
- **字体**：代码区域使用 `JetBrains Mono`，UI 文本使用 `IBM Plex Sans`
- **布局风格**：三栏布局（左侧配置树 + 右侧详情面板 + 顶部工具栏）
- **图标风格**：lucide-react 线性图标

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI 元素 |
|----------|----------|---------|
| 配置主页 | 文件上传区 | 拖拽上传区域，虚线边框，终端风格图标 |
| 配置主页 | 配置树面板 | 树形缩进列表，每行含复选框/三态框/输入框、配置名、类型标签、状态指示 |
| 配置主页 | 详情面板 | 卡片式布局，分区显示：基本信息、帮助文本、依赖链图 |
| 配置主页 | 工具栏 | 图标按钮组：搜索、展开/折叠、导出、重置 |
| 配置主页 | 搜索栏 | 顶部搜索输入框，实时过滤 + 高亮 |

### 4.3 响应式

- 桌面优先（1920×1080 为基准）
- 平板适配：详情面板改为底部抽屉
- 移动端：单栏模式，配置树全屏，详情面板为浮层

### 4.4 3D 场景

不适用。
