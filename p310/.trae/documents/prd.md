## 1. 产品概述

RTCP XR 呼叫质量分析平台——解析 RTCP Extended Report 报文，实时提取丢包率、抖动缓冲延迟、MOS 评分等关键 VoIP 质量指标，并通过可视化趋势图呈现呼叫质量变化。
- 面向网络运维工程师和 VoIP 质量监控人员，解决 RTCP XR 报文难以人工解读、质量趋势缺乏直观呈现的问题
- 帮助用户快速定位语音通话中的质量劣化时段与根因，提升故障响应效率

## 2. 核心功能

### 2.1 用户角色
| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 运维工程师 | 默认免登录 | 上传报文、查看解析结果与趋势图 |

### 2.2 功能模块
1. **仪表盘页面**：关键质量指标概览（最新丢包率、抖动延迟、MOS 分）、质量趋势折线图、报文上传与历史记录
2. **报文解析详情页面**：单条 RTCP XR 报文的完整解析结果展示，包含各 Report Block 的字段明细

### 2.3 页面详情
| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 仪表盘 | 质量概览卡片 | 展示最新丢包率、平均抖动延迟、当前 MOS 分三个核心指标卡片 |
| 仪表盘 | 质量趋势图 | 折线图展示丢包率、抖动延迟、MOS 分随时间的变化趋势，支持时间范围筛选 |
| 仪表盘 | 报文上传 | 支持上传十六进制/二进制 RTCP XR 报文文件，上传后自动解析并入库 |
| 仪表盘 | 历史记录表 | 展示最近解析的报文列表，含时间、SSRC、丢包率、MOS 等关键字段 |
| 报文详情 | Report Block 解析 | 展示 VoIP Metrics、Loss Report 等 Block 的逐字段解析结果 |

## 3. 核心流程

用户上传 RTCP XR 报文（十六进制文本或二进制文件）→ 后端解析报文提取 VoIP Metrics 等关键 Block → 计算/提取丢包率、抖动缓冲延迟、MOS 分 → 存储解析结果 → 前端请求趋势数据并渲染折线图

```mermaid
flowchart TD
    "A[用户上传RTCP XR报文]" --> "B[后端接收并解析报文]"
    "B" --> "C[提取VoIP Metrics Block]"
    "C" --> "D[获取丢包率/抖动延迟/MOS]"
    "D" --> "E[存储解析结果到SQLite]"
    "E" --> "F[前端请求趋势数据]"
    "F" --> "G[渲染质量趋势折线图]"
```

## 4. 用户界面设计

### 4.1 设计风格
- 主色调：深蓝灰（#1e293b）背景 + 翡翠绿（#10b981）强调色，传达专业监控氛围
- 辅助色：琥珀色（#f59e0b）用于警告/中等质量，红色（#ef4444）用于差质量/异常
- 按钮风格：圆角（rounded-lg），hover 微发光效果
- 字体：JetBrains Mono 用于数据展示，Source Sans 3 用于正文
- 布局：左侧导航栏 + 右侧主内容区，卡片式模块布局
- 图标：Lucide 图标库

### 4.2 页面设计概览
| 页面名称 | 模块名称 | UI 元素 |
|----------|----------|---------|
| 仪表盘 | 质量概览卡片 | 三列卡片，每卡片含图标+数值+趋势箭头，背景渐变 |
| 仪表盘 | 质量趋势图 | 多折线图（Chart.js），X轴时间，Y轴双轴（丢包率/延迟 + MOS），可切换指标 |
| 仪表盘 | 报文上传 | 拖拽上传区域 + 文本框粘贴十六进制，上传按钮 |
| 仪表盘 | 历史记录表 | 条纹表格，分页，点击行可跳转详情 |
| 报文详情 | Report Block 解析 | 折叠面板，每个 Block 一组，字段名-值对齐展示 |

### 4.3 响应式设计
- 桌面优先设计，平板和移动端自适应：卡片从三列变单列，表格横向滚动
