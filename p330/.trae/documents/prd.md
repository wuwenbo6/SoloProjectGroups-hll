## 1. 产品概述

基于LLDP协议的网络拓扑自动发现与可视化系统。后端通过Go语言监听并解析交换机发送的LLDP报文，提取设备ID、端口ID、TLV等关键信息；前端通过Canvas实时绘制网络拓扑图，直观展示网络设备间的连接关系。

- **核心问题**：网络运维人员需要手动梳理网络拓扑，效率低且容易出错
- **目标用户**：网络工程师、运维人员
- **产品价值**：自动发现网络拓扑、实时更新设备连接关系、可视化展示降低运维复杂度

## 2. 核心功能

### 2.1 功能模块

1. **拓扑总览页**：实时网络拓扑图（Canvas绘制）、设备节点与连线展示、缩放/拖拽交互
2. **设备详情页**：选中设备的LLDP详细信息、TLV字段解析、邻居设备列表

### 2.2 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|---------|---------|---------|
| 拓扑总览页 | 拓扑画布 | Canvas绘制设备节点与连线，支持缩放、拖拽、节点选中 |
| 拓扑总览页 | 工具栏 | 自动布局切换、刷新拓扑、全屏显示 |
| 拓扑总览页 | 设备列表侧栏 | 已发现设备列表，点击定位到拓扑图中的节点 |
| 拓扑总览页 | 状态栏 | 在线设备数、连接数、最后更新时间 |
| 设备详情页 | 设备信息卡 | Chassis ID、Port ID、System Name等LLDP核心字段 |
| 设备详情页 | TLV信息表 | 所有TLV类型及值的详细列表 |
| 设备详情页 | 邻居列表 | 该设备的直连邻居设备及连接端口信息 |

## 3. 核心流程

1. Go后端启动LLDP监听服务，捕获网络中的LLDP报文
2. 解析LLDP报文，提取Chassis ID、Port ID、TTL及自定义TLV
3. 根据设备ID和端口ID构建设备邻居关系
4. 通过WebSocket实时推送拓扑更新到前端
5. 前端接收数据后，自动在Canvas上绘制/更新拓扑节点和连线

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：深色科技风（#0a0e27 深空蓝背景，#00d4ff 霓虹蓝强调色）
- **辅助色**：#1a1f3a 卡片背景，#2a3050 边框，#4ade80 在线状态绿
- **按钮风格**：圆角微发光按钮，hover时发光增强
- **字体**：JetBrains Mono（数据展示），Noto Sans SC（中文内容）
- **布局风格**：左侧侧栏 + 中央Canvas画布 + 底部状态栏
- **图标风格**：线性科技感图标（lucide-react）

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|---------|---------|---------|
| 拓扑总览页 | 拓扑画布 | 深色背景、发光节点、流动连线动画、网格底纹 |
| 拓扑总览页 | 工具栏 | 半透明工具条、图标按钮、发光hover效果 |
| 拓扑总览页 | 设备列表侧栏 | 半透明面板、搜索框、设备列表项含状态指示灯 |
| 拓扑总览页 | 状态栏 | 底部固定、半透明、实时统计数据 |
| 设备详情页 | 设备信息卡 | 深色卡片、键值对布局、发光边框 |
| 设备详情页 | TLV信息表 | 深色表格、斑马纹、类型高亮 |

### 4.3 响应式设计

桌面优先设计，拓扑画布占满可用空间，侧栏可折叠以适配较小屏幕。
