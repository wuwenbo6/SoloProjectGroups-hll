## 1. 产品概述

IMU-RTK 传感器融合轨迹系统，通过扩展卡尔曼滤波（EKF）融合 IMU（加速度计/陀螺仪）和 RTK（高精度经纬度）数据，输出平滑、高精度运动轨迹，并在前端地图上实时可视化。

- 目标用户：无人机/机器人/自动驾驶研发人员，需要验证传感器融合算法效果
- 核心价值：提供端到端的传感器融合演示系统，从原始数据到平滑轨迹的全流程可视化

## 2. 核心功能

### 2.1 功能模块

1. **轨迹地图页**：实时地图轨迹显示、传感器数据面板、EKF 状态监控
2. **数据回放页**：历史轨迹回放、速度控制、数据导出

### 2.2 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 轨迹地图页 | 实时地图 | Leaflet 地图展示实时轨迹路径，支持缩放/拖拽 |
| 轨迹地图页 | 传感器面板 | 左侧面板展示 IMU 加速度/角速度实时曲线、RTK 定位点 |
| 轨迹地图页 | EKF 状态面板 | 右侧面板展示融合后的位置/速度/姿态及协方差 |
| 轨迹地图页 | 控制栏 | 启动/暂停数据流、重置 EKF、选择数据源 |
| 数据回放页 | 回放控制 | 播放/暂停/倍速/进度条 |
| 数据回放页 | 历史轨迹 | 地图上叠加显示完整历史轨迹 |

## 3. 核心流程

用户打开页面 → 后端模拟器生成 IMU+RTK 数据 → EKF 实时融合 → WebSocket 推送轨迹 → 前端地图实时绘制轨迹点和路径

```mermaid
flowchart LR
    "IMU 模拟器" --> "EKF 融合引擎"
    "RTK 模拟器" --> "EKF 融合引擎"
    "EKF 融合引擎" --> "WebSocket 推送"
    "WebSocket 推送" --> "前端地图渲染"
    "前端地图渲染" --> "用户查看轨迹"
```

## 4. 用户界面设计

### 4.1 设计风格

- 主色调：深蓝黑底 (#0a0e1a) + 青绿色荧光强调 (#00ffc8)
- 辅助色：暗灰面板 (#141926)、RTK 橙色标记 (#ff6b35)、IMU 蓝色线条 (#4d9fff)
- 按钮风格：圆角胶囊按钮，悬停发光效果
- 字体：JetBrains Mono（数据展示）+ Noto Sans SC（中文界面）
- 布局风格：全屏地图 + 左右悬浮面板 + 底部控制栏
- 图标：lucide-react 图标库

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI 元素 |
|----------|----------|---------|
| 轨迹地图页 | 实时地图 | 全屏 Leaflet 暗色瓦片地图，轨迹线渐变色，RTK 点橙色脉冲动画 |
| 轨迹地图页 | 传感器面板 | 左侧半透明玻璃面板，实时折线图（加速度3轴/角速度3轴），数值显示 |
| 轨迹地图页 | EKF 状态面板 | 右侧半透明面板，位置/速度/姿态数值 + 不确定性椭圆 |
| 轨迹地图页 | 控制栏 | 底部居中胶囊按钮组：播放/暂停/重置，数据源下拉 |
| 数据回放页 | 回放控制 | 底部播放条，倍速选择器，时间轴进度条 |
| 数据回放页 | 历史轨迹 | 全屏地图，完整轨迹线 + 当前回放点标记 |

### 4.3 响应式设计

- 桌面端优先：全屏地图 + 两侧面板
- 平板端：面板可折叠为底部抽屉
- 移动端：仅保留地图和底部控制栏，面板通过按钮切换
