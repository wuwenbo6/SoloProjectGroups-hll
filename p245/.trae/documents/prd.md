## 1. 产品概述

J1939 TP协议模拟器是一款面向汽车电子工程师和嵌入式开发者的协议可视化调试工具。基于Python + socketcan模拟SAE J1939传输层协议（Transport Protocol），将大于8字节的J1939消息拆分为多个CAN帧进行传输（BAM广播模式与CMDT点对点模式），并通过Web前端实时展示消息的拆包、传输、重组全过程，以及丢包与重传机制的运作方式。

- 核心目标：帮助工程师直观理解J1939 TP协议的分帧逻辑、BAM与CMDT两种传输模式的差异、以及丢包重传的完整流程
- 目标用户：汽车电子工程师、嵌入式开发者、CAN总线协议学习者

## 3. 核心流程

### 3.1 BAM广播模式流程

操作员选择BAM模式 → 配置消息参数 → 发送BAM公告帧（PGN EC00） → 按间隔逐帧广播数据帧（PGN EB00） → 所有节点接收并重组 → 重组完成显示完整消息

### 3.2 CMDT点对点模式流程

操作员选择CMDT模式 → 配置消息参数与目标地址 → 发送RTS帧（PGN EC00） → 接收端回复CTS帧 → 按CTS窗口发送数据帧（PGN EB00） → 若丢包则CTS请求重传 → 全部到达后发送EndOfMsgAck

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：深色工业风背景（#0a0e17），科技蓝强调色（#00d4ff），警示红（#ff3b5c）标记丢包，成功绿（#00ff88）标记重组完成
- **按钮风格**：圆角矩形，hover时发光边框，工业控制台质感
- **字体**：数据区域使用等宽字体（JetBrains Mono），标题使用科技感字体（Orbitron）
- **布局风格**：左侧控制面板 + 右侧主可视化区域，顶部状态栏
- **图标风格**：线性图标（Lucide），与科技主题一致

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 控制台 | 模式选择 | 两个高亮卡片，点击切换BAM/CMDT，选中态发光边框 |
| 控制台 | 参数配置 | 表单输入框组，滑块控制丢包率，数字输入框控制消息大小/地址 |
| 控制台 | 模拟控制 | 发光启动/暂停/重置按钮，状态指示灯 |
| 协议可视化 | 帧传输时间线 | 垂直时间轴，帧节点按时间排列，点击展开帧详情 |
| 协议可视化 | 消息重组面板 | 进度条+帧方块网格，已接收帧高亮，缺失帧灰暗 |
| 协议可视化 | 丢包重传事件 | 丢包帧红色闪烁标记，重传帧黄色标记，CTS请求箭头动画 |
| 协议可视化 | 协议状态机 | 状态节点圆形图，流转箭头动画，当前状态高亮 |

### 4.3 响应式设计

- 桌面优先设计，最小宽度1024px
- 控制面板可折叠以适应较小屏幕
- 可视化区域支持滚动浏览长消息
