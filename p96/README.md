# GNSS 欺骗检测系统

一个基于Electron的GPS信号分析和欺骗检测应用程序。

## 功能特性

- 🛰️ **实时GPS信号分析** - 集成GNSS-SDR库进行专业级信号处理
- 🚨 **欺骗检测** - 检测伪距突变、信号功率异常、异常稳定等欺骗特征
- 📊 **卫星可视化** - 天空视图、信号功率趋势图、伪距变化曲线
- 📡 **星历数据显示** - 完整的卫星轨道参数展示
- 🔔 **实时报警系统** - 桌面通知、声音警报、窗口闪烁
- 💾 **数据库存储** - SQLite持久化存储卫星数据和异常记录

## 欺骗检测算法

### 多径抑制（城市峡谷优化）
- **多径特征分类**：基于SNR波动模式、仰角、锁相环丢失事件、自相关性综合判断
- **置信度评估**：多径置信度>50%时自动抑制信号功率异常误报
- **降级机制**：确认多径影响的异常自动降低严重程度
- **低仰角修正**：<30°仰角卫星自动增加多径置信度

### 渐进式欺骗检测（解决缓慢变化漏检）

#### 1. CUSUM控制图（累积和）
- 检测微小但持续的伪距漂移
- 对缓慢变化的欺骗攻击敏感
- 参数：阈值15，漂移2σ

#### 2. EWMA控制图（指数加权移动平均）
- 检测信号功率的渐进变化
- 自适应权重，近期数据影响更大
- 参数：α=0.15，3σ控制限

### 传统突变检测
- **伪距突变**：>50米突变
- **信号功率突升**：>15 dB-Hz
- **信号功率突降**：>10 dB-Hz
- **载噪比过低**：<35 dB-Hz

### 交叉验证检测
- **卫星间一致性检查**：单颗卫星伪距与其他卫星Z-score>2.5时报警
- **SNR均匀性检测**：多颗卫星SNR异常一致（标准差<2dB）暗示欺骗攻击

### 异常稳定性检测
- 检测信号异常平稳（标准差<0.8）
- 自然信号应有一定噪声特性

## 安装和运行

### 前置要求

- Node.js 18+
- npm 或 yarn
- GNSS-SDR（可选，用于真实信号处理）

### 安装依赖

```bash
npm install
```

如果遇到权限问题：
```bash
npm_config_cache=/tmp/npm_cache npm install
```

### 运行应用

开发模式（带开发者工具）：
```bash
npm run dev
```

生产模式：
```bash
npm start
```

### 构建应用

```bash
npm run build
```

## 使用说明

### 1. 启动分析
点击"开始分析"按钮启动GPS信号分析系统。

### 2. 查看卫星
- **左侧面板**：显示当前可见卫星列表和SNR值
- **天空视图**：极坐标显示卫星方位角和仰角
- **颜色编码**：绿色(>40dB)、黄色(30-40dB)、红色(<30dB)

### 3. 信号分析
- 切换到"信号分析"标签页
- 查看SNR趋势图和伪距变化曲线

### 4. 异常处理
- 检测到异常时自动弹出报警窗口
- 在"异常记录"标签页查看所有异常
- 点击"确认"按钮消除警报
- 支持按类型和严重程度筛选

## 数据库

应用使用SQLite存储以下数据：

- `satellites` - 卫星基本信息
- `ephemeris` - 卫星星历数据
- `signal_history` - 信号历史数据
- `anomalies` - 异常事件记录
- `system_logs` - 系统日志

## 模拟模式

默认运行在模拟模式，会：
- 生成8颗模拟GPS卫星
- 注入随机噪声模拟真实信号
- 5%概率随机注入欺骗事件用于测试

## 切换到真实GNSS-SDR

1. 安装GNSS-SDR：
   ```bash
   # macOS
   brew install gnss-sdr
   
   # Ubuntu
   sudo apt install gnss-sdr
   ```

2. 修改 `src/main/gnssAnalyzer.js` 中的配置：
   ```javascript
   simulationMode: false
   ```

3. 配置信号源：
   编辑 `config/gnss-sdr.conf` 设置正确的信号源路径或SDR设备。

## 技术栈

- **Electron 28** - 跨平台桌面应用框架
- **better-sqlite3** - 高性能SQLite数据库
- **GNSS-SDR** - 软件定义无线电GPS处理
- **Canvas API** - 数据可视化

## 注意事项

1. 首次运行需要下载Electron二进制文件，可能需要较长时间
2. better-sqlite3需要编译原生模块，确保有编译环境
3. GNSS-SDR集成需要单独安装且仅支持Linux/macOS
