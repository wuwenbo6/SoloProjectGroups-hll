# HLS资源估算Web应用 - 产品需求文档

## 1. 产品概述

### 1.1 产品定位
面向FPGA开发者的HLS（高层次综合）资源估算工具，帮助开发者在合成前快速预估C代码转换为硬件后的资源占用情况。

### 1.2 目标用户
- FPGA工程师
- 硬件加速开发者
- 嵌入式系统设计师
- 计算机体系结构研究者

### 1.3 核心价值
- **快速反馈**：无需等待完整HLS合成流程，秒级获取资源估算
- **历史追踪**：记录所有估算历史，方便对比不同版本的优化效果
- **优化建议**：智能分析代码，提供资源优化方向
- **可视化**：直观的柱状图展示LUT、DSP、BRAM资源占用

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 代码上传与编辑
- 支持C代码文件上传（.c, .cpp, .h）
- 内置代码编辑器，支持语法高亮
- 支持直接粘贴代码

#### 2.1.2 资源估算
- LUT（查找表）资源估算
- DSP（数字信号处理单元）资源估算
- BRAM（块RAM）资源估算
- 柱状图可视化展示
- 资源使用率百分比显示

#### 2.1.3 优化建议
- 循环优化建议（流水线、展开）
- 数组分区建议
- 数据类型优化建议
- 函数内联建议

#### 2.1.4 历史记录
- 存储所有估算历史
- 支持按时间、代码名称搜索
- 支持查看历史详情
- 支持历史记录删除

### 2.2 用户流程

```
用户上传/粘贴C代码
    ↓
点击"开始估算"
    ↓
后端调用HLS估算引擎
    ↓
返回资源数据和优化建议
    ↓
前端展示柱状图和建议
    ↓
自动保存到历史记录
```

## 3. 非功能需求

### 3.1 性能要求
- 单次估算响应时间 < 3秒
- 支持同时10个并发用户
- 历史记录查询响应 < 500ms

### 3.2 技术要求
- 响应式设计，支持桌面端和移动端
- 代码编辑器支持语法高亮
- 图表支持缩放和交互

### 3.3 数据安全
- 本地数据库存储，数据不上传云端
- 支持导出历史记录

## 4. 界面设计

### 4.1 主界面布局
- **左侧**：代码编辑器区域
- **右侧**：资源估算结果 + 优化建议
- **底部**：历史记录列表

### 4.2 视觉风格
- 技术感深色主题
- 代码编辑器使用Monaco风格
- 柱状图使用蓝绿渐变色系
- 清晰的信息层级

## 5. 技术选型

### 5.1 前端
- 框架：React 18
- 构建工具：Vite
- 代码编辑器：Monaco Editor
- 图表库：Chart.js
- 样式：Tailwind CSS

### 5.2 后端
- 运行时：Node.js
- 框架：Express
- HLS估算：模拟引擎（可扩展支持真实HLS工具）

### 5.3 数据库
- SQLite（轻量级，本地存储）
